# 题库生成规则

本文档用于指导 GPT、Claude、Cursor 等工具为当前刷题工程生成新题库。

生成结果必须能直接粘贴到 `src/data/sampleQuestions.ts` 的 `sampleQuestions` 数组中。

## 1. 输出格式

每道题必须是一个 TypeScript 对象，整体必须是 `Question[]` 数组的一部分。

字段必须完整：

```ts
{
  id: "dl-011",
  type: "single",
  category: "Transformer",
  tags: ["Self-Attention", "QKV"],
  difficulty: "medium",
  question: "题干文本",
  options: {
    A: "选项 A",
    B: "选项 B",
    C: "选项 C",
    D: "选项 D"
  },
  answer: ["B"],
  explanation: "答案解析文本"
}
```

## 2. 字段规则

### id

- 必须是字符串。
- 必须全局唯一，不能和已有题目重复。
- 推荐格式：`dl-001`、`dl-002`、`ml-001`、`python-001`。
- 同一批题目编号必须连续。

### type

只允许两个值：

```ts
"single" | "multiple"
```

- `single` 表示单选题。
- `multiple` 表示多选题。

### category

- 必须是字符串。
- 用于首页分类统计。
- 同一知识领域应尽量复用同一个分类名。
- 示例：`神经网络基础`、`Transformer`、`优化器`、`模型评估`。

### tags

- 必须是字符串数组。
- 建议 1 到 4 个标签。
- 标签应体现具体知识点，不要太泛。
- 好例子：`["BatchNorm", "训练阶段", "推理阶段"]`
- 差例子：`["AI", "基础", "知识"]`

### difficulty

只允许三个值：

```ts
"easy" | "medium" | "hard"
```

- `easy`：概念识别、基础定义。
- `medium`：机制理解、常见坑点、对比分析。
- `hard`：边界条件、推理细节、工程取舍。

### question

- 必须是清晰的问题句。
- 不要出现“以上说法都对”这类依赖选项顺序的题干。
- 不要让题干和答案存在歧义。
- 多选题必须明确提示“以下哪些”。

### options

- key 只允许使用 `A`、`B`、`C`、`D`、`E`、`F`。
- 至少 4 个选项。
- 一般使用 4 个选项，复杂多选可以用 5 到 6 个。
- 选项内容必须互斥或边界清晰。
- 不要使用“以上都对”“以上都不对”，除非题目确实需要考查这种形式。

### answer

- 必须是数组。
- 单选题也必须写数组，例如：

```ts
answer: ["B"]
```

- 多选题写多个正确选项，例如：

```ts
answer: ["A", "C", "D"]
```

- `answer` 中的选项必须存在于 `options`。
- `single` 只能有 1 个答案。
- `multiple` 必须至少有 2 个答案。

### explanation

- 必须解释为什么正确答案正确。
- 必须顺带说明关键错误选项为什么不对。
- 不要只写“答案是 B”。
- 建议 80 到 220 字。
- 对多选题，应解释每个正确选项的依据。

## 3. 内容质量要求

生成题目时必须遵守：

- 题目必须考查明确知识点，不要考语言游戏。
- 错误选项要有迷惑性，但不能错误得离谱。
- 避免重复考同一个表述。
- 避免题干泄露答案。
- 避免答案分布过于集中，例如连续大量都是 `B`。
- 多选题不要把所有选项都设为正确。
- 多选题不要只有一个正确选项。
- 解析必须能帮助复习，不只是判断对错。

## 4. TypeScript 兼容要求

输出必须符合当前工程类型：

```ts
export type QuestionType = "single" | "multiple";
export type OptionKey = "A" | "B" | "C" | "D" | "E" | "F";
export type Difficulty = "easy" | "medium" | "hard";

export interface Question {
  id: string;
  type: QuestionType;
  category: string;
  tags: string[];
  difficulty: Difficulty;
  question: string;
  options: Partial<Record<OptionKey, string>>;
  answer: OptionKey[];
  explanation: string;
}
```

不能输出 JSON 专用格式。

错误示例：

```json
{
  "id": "dl-011",
  "type": "single"
}
```

正确示例：

```ts
{
  id: "dl-011",
  type: "single"
}
```

## 5. 推荐生成比例

一批 20 道题建议：

- 单选题：12 到 14 道。
- 多选题：6 到 8 道。
- `easy`：4 到 6 道。
- `medium`：10 到 12 道。
- `hard`：2 到 4 道。

如用于快速入门，可以提高 `easy` 比例。

如用于面试冲刺，可以提高 `medium` 和 `hard` 比例。

## 6. 生成前必须指定的信息

让 GPT 生成题库前，最好明确这些信息：

- 题库主题，例如：`深度学习`、`机器学习`、`Python`、`操作系统`。
- 题目数量。
- 起始 ID，例如：`dl-011`。
- 单选和多选比例。
- 难度分布。
- 是否偏面试、考试、工程实践或概念复习。

## 7. 可直接复制的提示词

```txt
请为我的本地刷题工程生成一批题库。

主题：深度学习
题目数量：20
起始 id：dl-011
题型比例：单选 14 道，多选 6 道
难度比例：easy 5 道，medium 12 道，hard 3 道
风格：偏面试八股，但解析要有实际理解价值

输出要求：
1. 只输出 TypeScript 对象数组片段，不要输出 Markdown。
2. 不要包含 import，不要包含 export const。
3. 每道题必须包含 id、type、category、tags、difficulty、question、options、answer、explanation。
4. type 只能是 "single" 或 "multiple"。
5. difficulty 只能是 "easy"、"medium"、"hard"。
6. options 的 key 只能使用 A、B、C、D、E、F。
7. answer 必须是数组，单选题也要写成 ["B"]。
8. single 只能有 1 个正确答案。
9. multiple 至少有 2 个正确答案。
10. answer 中的选项必须存在于 options。
11. 每题至少 4 个选项。
12. 解析必须说明为什么正确，以及关键错误选项为什么错。
13. 不要使用“以上都对”“以上都不对”。
14. id 不能重复，并从 dl-011 开始连续编号。

输出格式示例：
[
  {
    id: "dl-011",
    type: "single",
    category: "Transformer",
    tags: ["Self-Attention", "QKV"],
    difficulty: "medium",
    question: "Transformer 中 Self-Attention 的主要作用是什么？",
    options: {
      A: "只提取局部边缘特征",
      B: "建模序列中不同位置之间的依赖关系",
      C: "替代所有损失函数",
      D: "只能用于图像分类任务"
    },
    answer: ["B"],
    explanation: "Self-Attention 通过 Query、Key、Value 计算 token 之间的相关性，用于建模序列内部不同位置之间的依赖关系。A 更接近 CNN 的局部特征提取，C 和 D 都不符合 Transformer 的实际用途。"
  }
]
```

## 8. 粘贴到工程的方式

生成后打开：

```txt
src/data/sampleQuestions.ts
```

把新题目对象追加到：

```ts
export const sampleQuestions: Question[] = [
  // 旧题目
  // 新题目粘贴到这里
];
```

如果 GPT 输出的是完整数组：

```ts
[
  {
    id: "dl-011"
  }
]
```

只复制数组里面的对象，不要复制最外层 `[` 和 `]`。

## 9. 自检清单

粘贴前检查：

- `id` 没有重复。
- 每个对象之间有英文逗号。
- 最后一题后面可以有逗号，也可以没有。
- 字符串使用英文双引号。
- `answer` 是数组。
- 多选题答案数量至少 2 个。
- `answer` 中没有不存在的选项。
- `type`、`difficulty` 没有拼写错误。
- 没有 Markdown 代码围栏。

## 10. 常见错误

错误：单选题答案不是数组。

```ts
answer: "B"
```

正确：

```ts
answer: ["B"]
```

错误：多选题只有一个答案。

```ts
type: "multiple",
answer: ["C"]
```

正确：

```ts
type: "multiple",
answer: ["A", "C"]
```

错误：答案选项不存在。

```ts
options: {
  A: "选项 A",
  B: "选项 B",
  C: "选项 C",
  D: "选项 D"
},
answer: ["E"]
```

正确：要么把答案改成已存在选项，要么补充 `E` 选项。
